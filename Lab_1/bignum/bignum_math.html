2.2 Your task

For this lab, you will write a program that can perform addition and subtraction (not multiplication or division) of large numbers of any base between 2 and 36 (2 and 36 inclusive). You will have to use arrays instead of any regular datatypes, in order to be able to hold these large numbers.

Starting from the given file, you’ll need to fill in the function perform_math that will take in the inputs from main, as well as the operation to be performed. You will return a result from your function, and main will print out this result when finished.

But wait; there are two other functions (valid_input and bignum_print) that you will need to complete. valid_input checks the input array of characters and determines whether the input is well formed and in the correct base and sign. bignum_print takes an array of integers for the result and correctly prints it to the terminal in the given base.

In an attempt to make your life easier, we have provided a function that converts the input array of characters to an array of integers (Notice that this is called in main, after the input array of characters has been determined to be well formed by valid_input). Therefore perform_math, will be given two arrays of int. Each index of an array will contain a number (integer) for each character in a given input.

Because arrays in C are not complex objects, there is no good way to set or compute an array’s length. Thus, we’ll need some way to denote that the number is over. Since we actually have ints, a negative number such as -1 (negative one) will indicate that the number is finished. Thus, if I input the number 1234567ABC, the array that gets passed to perform_math will be { 1, 2, 3, 4, 5, 6, 7, 10, 11, 12, -1 }.

Also, we don't allow negative inputs.

When you return your result, you will use the same format (so take care of the negative results in case of subtraction).

Here is a sample compilation and run-through:

gcc -Wall -o bignum_math bignum_math.c
./bignum_math 16 f4d905 + F
Result: F4D914

In order to get you started in this lab, we have provided a function add which adds two numbers correctly. You may refer to it as an example before creating the correct Subtraction function.

3 Tips

  • It is perfectly legal to use any code that we present in discussion or lecture. We may supply you with some helpful code hints.

  • It is very easy to write code that will accidentally run off the end of an array in C. Unlike Java, there will be no compilation error, no warning, not even necessarily a run-time error. The most likely thing you will encounter is strange behavior that doesn’t make any sense. Double-check your array accesses to avoid this problem!
